## Anti-pattern verdict

**Pass.** The charcoal, bone, lime, and 3D archive identity remains recognizable, while the repeated AI-concept signals were removed: no animated or static grain, no bounce motion, no layout-property animation, no full-row lime glow, and no checkless selection wall. The remaining 3D object functions as product identity rather than generic generated decoration.

**Deterministic scan:** 0 warnings, down from 3. Strict false positives: 0.

**Browser evidence:** Custom headless Chrome was used after the native in-app browser backend was unavailable. At 1440px, selected source rows had a restrained `rgba(217, 255, 63, 0.043)` tint, no shadow, a visible check marker, and dark native-control color scheme. At 400px, document overflow was 0 and sampled mode chips, problem checkboxes, delete controls, and selects reached 44px height. Mutable detector overlays were unavailable.

## What improved

1. SourcePicker, quiz selection, and exam completion now use visible check or indeterminate markers plus quiet tint instead of full lime fill and glow.
2. Quiz generation uses an exclusion set, preserving user choices while including newly ready material by default.
3. Quiz conditions and AI generation have persistent visual labels; progress, icon buttons, mode chips, status marks, and expandable questions have semantic state or names.
4. Coarse-pointer targets, keyboard-openable subject rows, focus-visible delete actions, small-text contrast, and long-title wrapping close the accessibility boundary gaps.
5. Grain, bounce, cursor geometry reflow, and subject-card padding animation were removed without adding a new abstraction or dependency.
