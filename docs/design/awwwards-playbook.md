# Awwwards-급 디자인 플레이북 (StudyWork 적용판)

작성: 2026-07-10. 근거: 딥 리서치(소스 21개, 주장 105개 중 교차검증 통과 16개) + 검증된 실무 기법.
표기: ✅ = 3표 교차검증 통과 사실, ⚙️ = 업계 표준 실무 기법(검증 표본 부족하나 소스 다수).

## 0. 심사가 실제로 보는 것

- ✅ Awwwards 채점 가중치: **Design 40% · Usability 30% · Creativity 20% · Content 10%** — 비주얼+사용성이 70%. 6.5점 이상 Honorable Mention, 당일 최고점이 SOTD. 심사위원 최소 18명, 이상치 점수 자동 제거.
- ✅ 수상작 공통 형질: **거대한 타이포 · 페이지 전환/로더/스태거 애니메이션 · 커서 인터랙션 · 패럴랙스/스무스 스크롤 · 3D/셰이더 · 리로드마다 조금씩 다른 랜덤성** — 단, 전부 필수는 아님.
- ✅ 25개 수상작 분석 공통점: 장식이 아니라 **목적 있는** 선택, 모든 기기에서 동작, 접근성 유지, UX를 방해하지 않는 디자인.
- **교훈**: 3D·모션은 Usability 30%를 깎아먹는 순간 역효과. "화려함"은 콘텐츠 읽기를 방해하지 않는 선까지.

## 1. 타이포그래피 — 레이아웃의 주인공

- ✅ Awwwards가 공식 트렌드로 명명: **타이포가 레이아웃의 중심 요소** — 그리드 기반을 벗어나 인쇄 에디토리얼에 가까운 구성.
- ⚙️ 스케일 대비: 디스플레이 `clamp(48px, 8~12vw, 160px)` vs 본문 15~16px — **8~10배 대비**가 에디토리얼 인상의 핵심. 중간 크기를 남발하지 않는다(대·소 2계층 + 마이크로 라벨).
- ⚙️ 마이크로 라벨: 10~11px, `letter-spacing: 0.15~0.25em`, uppercase, dim 컬러 — 섹션 인덱스("01 — ARCHIVE")로 사용.
- ⚙️ 페어링: 개성 있는 디스플레이(세리프 또는 확장형 그로테스크) + 중립 본문 1개. 3개 이상 금지.
- **라인 마스크 리빌**: ✅ GSAP SplitText는 `mask: "lines"` 옵션으로 각 줄을 클리핑 래퍼로 감싸 한 옵션으로 구현(chars/words/lines 배열 노출). ⚙️ 의존성 없이 하려면: 줄을 `overflow:hidden` 래퍼로 감싸고 내부 요소를 `translateY(110%) → 0`, 줄당 60~90ms 스태거.

## 2. 3D / WebGL — 오브제는 하나, 규율은 엄격하게

- ✅ 수상작 지배 스택: **Three.js/WebGL**(3D) + **GSAP/ScrollTrigger**(스크롤 연동). 성능은 "나중에 최적화"가 아니라 **설계 제약으로 day one부터**.
- **글래스/트랜스미션 머티리얼 검증 레시피** ✅:
  ```
  transmission: 1, roughness: 0, thickness: 0.2, ior: 1.2,
  chromaticAberration: 0.02, backside: true   // drei MeshTransmissionMaterial
  ```
- r3f 성능 규율:
  - ✅ `<Canvas frameloop="demand">` — 변화 있을 때만 렌더(아이패드 배터리·발열에 직결). 상시 회전 오브제면 invalidate 루프를 직접 제어.
  - ✅ 드로우콜 예산: **최대 1000, 목표 수백 이하**(메시 1개=콜 1개). 반복 요소는 instancedMesh.
  - ✅ `state.performance.regress()` — 인터랙션 중 품질 강등, 200ms 디바운스 후 자동 복구 (drei PerformanceMonitor/AdaptiveDpr와 결합).
  - ✅ 실증 예산: 프로덕션급 데모(SINGULARITY)가 메시 27·텍스처 184·셰이더 49·40k 트라이앵글 + 물리·후처리를 **총 에셋 2.1MB**로 안정 구동 — "무겁게"의 상한이 생각보다 낮다.
  - ✅ glTF 에셋은 `gltfjsx -S -T -t`(단순화+텍스처 변환)로 90% 감량 사례.
  - ⚙️ DPR은 1~1.5로 캡, 모바일에서 후처리 끔.
- **통합 패턴** ⚙️: (a) 고정 배경 히어로 오브제 — 마우스에 lerp 추종, 상태(페이지/탭)에 따라 위치·왜곡 변화 (b) 스크롤 내러티브 — 긴 랜딩용. **앱에는 (a)가 맞다** — 콘텐츠 위 오버레이 금지, pointer-events 차단.

## 3. 모션 언어 (수치)

- ⚙️ 이징: UI 표준 `cubic-bezier(0.22, 1, 0.36, 1)`(≈expo-out 계열), 등장 0.6~1.0s(히어로)/0.3~0.45s(UI), 퇴장은 등장의 60~70%.
- ⚙️ 스태거: 텍스트 줄 60~90ms, 카드/리스트 40~80ms, 6개 이상이면 총합이 600ms 넘지 않게 감쇠.
- ⚙️ 페이지/탭 전환: 콘텐츠 페이드+8~16px 상승, 또는 마스크 와이프. 300~500ms. 로더는 첫 진입 1회만.
- ✅ "리로드마다 조금씩 다른" 랜덤성 — 오브제 초기 회전각·파티클 시드 등 미세 변주.
- ⚙️ `prefers-reduced-motion` 존중 — 수상작 공통(접근성 항목).

## 4. 레이아웃 — 비대칭 에디토리얼

- ✅ 인쇄 에디토리얼 지향(타이포 중심 구성). ⚙️ 구체 기법:
  - 좌측 정렬 위계 + 큰 여백(섹션 상하 96~160px, 데스크톱 기준).
  - **1px 헤어라인**(`rgba(text, .12~.16)`)으로 영역 구분 — 박스/카드 남발 대신 룰과 여백.
  - 오버사이즈 요소의 의도적 클리핑(인덱스 넘버가 화면 밖으로 잘려나감), 요소 겹침(넘버 위에 텍스트).
  - 그리드는 12칸을 깔되 **비대칭 배분**(3/9, 2/7/3) — 중앙 정렬 히어로 + 3열 카드는 전형적 템플릿 냄새.

## 5. 컬러 규율

- ⚙️ 베이스 1 + 텍스트 1 + **액센트 정확히 1** (+상태색). 액센트는 전체 픽셀의 5% 미만 — 인덱스·활성 상태·핵심 CTA·3D 림라이트에만.
- ⚙️ 순수 #000/#fff 대신 웜/쿨 틴트(예: #0e0e10, #EDEAE0). 그레인·노이즈 텍스처는 flat함을 깨는 저비용 장치.

## 6. 마이크로 인터랙션

- ⚙️ 커스텀 커서: 4~8px 점 + 24~40px 링, 링은 `lerp(0.12~0.2)`로 지연 추종, 인터랙티브 호버 시 scale 2~3배. 터치 기기에선 완전 비활성.
- ⚙️ 마그네틱 버튼: 요소 중심↔커서 벡터의 20~35%만큼 translate, 이탈 시 elastic 복귀(Codrops 표준 패턴). 주요 CTA에만.
- ⚙️ 호버: 언더라인 드로우(왼→오 scaleX), 텍스트 슬라이드 스왑(위로 밀며 복제 텍스트 등장). 글로우 박스섀도는 쓰지 않는다.

## 7. "AI 생성 티" 안티패턴 (제거 대상)

⚙️ 슬롭 가이드들의 공통 지적 + 이 프로젝트에서 걷어낼 것:

| 안티패턴 | 대체 |
|---|---|
| 보라/네온 그라데이션, 글로우 박스섀도 | 단일 액센트 + 헤어라인 |
| 모든 요소 20px+ 라운드 + 글래스모피즘 카드 | 플랫 패널, 라운드 6~10px, 1px 보더 |
| 중앙 정렬 히어로 + 균등 3열 기능 카드 | 비대칭 에디토리얼, 타이포 중심 |
| Inter/Roboto + 균일한 폰트 크기 | 개성 디스플레이 + 8배 스케일 대비 |
| 마우스 따라다니는 글로우 스포트라이트 | 커서 점+링, 마그네틱 |
| 이모지 아이콘, 균일 패딩 카드 그리드 | 인덱스 넘버·마이크로 라벨·룰 |
| 모든 요소에 균등한 모션 | 위계적 모션(히어로만 크게, UI는 즉답) |

## 8. StudyWork 적용 맵

- **팔레트**: 차콜 `#0e0e10` / 본 `#EDEAE0` / 액센트 애시드 라임 `#D9FF3F` 1개. 기존 시안·민트·글로우 전부 제거.
- **타이포**: 디스플레이(영문·숫자) = 개성 그로테스크 1종(Syne/Unbounded 계열, Inter·Space Grotesk 금지), 한글 헤드 = Gowun Batang 대형, 본문 = Pretendard. 탭 = "01 채팅 … 05 노트" 인덱스형.
- **3D**: r3f 고정 배경 Canvas 1개 — 흑요석/글래스 오브제(위 레시피), 라임 림라이트, 마우스 lerp 추종, 페이지별 무드(login=중앙 대형, subjects=우측 표류, detail=축소·후퇴). frameloop 제어 + DPR 캡 + reduced-motion 대응. React.lazy로 지연 로드.
- **모션**: 탭 전환 350ms 페이드+상승, 헤드라인 라인 마스크 리빌(줄당 80ms), 리스트 스태거 50ms.
- **유지**: 기능·API·건조한 카피·그레인. 인쇄 스타일시트는 라이트 세리프 유지.
- **성능 가드(아이패드)**: 드로우콜 < 100, 에셋 < 2MB, 후처리 최소, 60fps 미달 시 regress.
