# StudyWork 설계 문서

날짜: 2026-07-08

## 개요

내 수업 자료(사진·PDF·텍스트)를 과목별로 쌓아두면, 그 자료를 기반으로 설명하고, 퀴즈를 내고, 오답을 분석하고, 시험 계획까지 짜주는 **혼자 쓰는 개인 공부 비서 웹 앱**. 폰·PC 어디서든 URL로 접속한다.

- 사용자: 1명 (본인)
- AI 백엔드: **Claude Agent SDK (Max 플랜 구독 사용, API 키·추가 과금 없음)** — 2026-07-08 로컬 방식으로 전환
- 호스팅: **로컬 (내 맥에서 Node 서버 실행)**. 폰은 같은 와이파이에서 맥 IP로 접속, 외부는 Tailscale 등으로 확장 가능
- (이전 결정: Cloudflare Workers + D1 + R2 + Claude API — API 과금을 피하기 위해 로컬로 전환. git 히스토리에 보존)

## 기능 (단계별 구현)

각 단계가 끝나면 그 시점에서 바로 쓸 수 있는 상태가 되도록 한다.

### 1단계 — MVP: 과목 + 자료 + 튜터 채팅
- 과목 생성/관리 (과목마다 고유 색상)
- 자료 업로드: 사진(필기·프린트·교과서), PDF, 직접 타이핑/붙여넣기
- 업로드 시 Claude가 사진/PDF에서 텍스트를 추출·정리해 DB에 저장
- 과목별 AI 채팅: 해당 과목 자료가 자동으로 컨텍스트에 포함되어 "내 수업 맞춤" 답변
- **단권화**: 버튼 하나로 과목의 모든 자료를 하나의 통합 정리 노트로 생성 — 가독성 좋은 마크다운 구조, 중요 부분 강조 표시, 내용 누락 금지(중복만 병합). 재생성 가능, 과목당 노트 1개 유지

### 2단계 — 퀴즈
- **출제 소스 선택**: ① 업로드한 문제(문제집·프린트를 파일/사진으로 올리면 Claude가 문항·선택지·정답·풀이로 추출·구조화) ② AI 생성 문제(자료 기반 객관식·단답·OX) — 둘 중 선택하거나 섞어서 출제
- **난이도 선택**: 문제 생성·출제 시 난이도(하/중/상) 지정. 추출한 문제도 Claude가 난이도 태깅
- 풀기 + 즉시 채점·해설
- 틀린 문제 위주 반복 출제
- **PDF 다운로드**: 선택한 문제들(또는 오답 모음)을 PDF로 내보내기 — 문제지/정답지 분리

### 3단계 — 오답 노트
- 퀴즈에서 틀린 문제 자동 축적
- 학교 시험·문제집에서 틀린 문제를 사진으로 등록
- 유형·약점 분석, 약점 위주 복습 모드

### 4단계 — 시험 계획
- 시험일·범위 입력 → 공부 계획 생성
- 진도 체크, 남은 날짜 기준 계획 조정

## 아키텍처 (로컬)

```
폰/PC 브라우저 (React + Vite SPA)
   │  (같은 와이파이: http://<맥IP>:8787)
   ▼
내 맥의 Node 서버 (Hono, 정적 프론트 서빙 + REST API)
   ├─ SQLite 파일 : 과목, 자료, 채팅, 노트 (better-sqlite3, D1 호환 어댑터)
   ├─ 로컬 폴더   : 사진·PDF 원본 파일 (R2 호환 어댑터)
   └─ Claude Agent SDK : 텍스트 추출(Read 툴로 이미지/PDF 직접 읽기), 채팅, 단권화
        └─ 맥의 Claude Code 로그인(Max 플랜) 인증을 그대로 사용 — API 키 불필요
```

- 라우트·스키마·프론트엔드는 Cloudflare 버전에서 그대로 유지, 저장소·AI 호출부만 어댑터로 교체

### 자료 처리 흐름 (RAG-lite)
1. 업로드 → 원본을 R2에 저장
2. Claude로 텍스트 추출·정리 → D1 `materials`에 텍스트 저장
3. 채팅/퀴즈 생성 시 해당 과목의 자료 텍스트를 프롬프트 컨텍스트로 주입
4. 자료가 많아져 컨텍스트 한도를 넘으면 추후 Vectorize(벡터 검색)를 끼워 넣을 수 있도록 자료 조회를 별도 모듈로 분리

### 데이터 모델 (D1)
- `subjects` — id, 이름, 색상, 생성일
- `materials` — id, subject_id, 종류(사진/PDF/텍스트), R2 키, 추출 텍스트, 제목, 생성일
- `chats` / `messages` — 과목별 대화 기록
- `quizzes` / `questions` / `attempts` — 퀴즈, 문항, 풀이 기록 (2단계)
- `wrong_answers` — 오답 (출처: 퀴즈 자동/사진 등록), 유형 태그 (3단계)
- `exams` / `study_plans` — 시험 일정, 계획 항목 (4단계)

### 인증
- 간단한 비밀번호 1개 → 검증 후 토큰 발급(쿠키). 혼자 쓰는 앱이지만 API 비용 보호를 위해 필수.

### 비용 가드
- 하루 Claude 호출 횟수 상한 (D1 카운터, 기본 200회 — 설정에서 변경 가능). 초과 시 명확한 안내 메시지.

## UI 디자인 방향 — "개념 별자리" (다크, 시네마틱)

데스크톱 웹사이트 우선. 레퍼런스(토스 산통점)는 무드 참고만, 컨셉은 자체 아이덴티티: 개념 = 별, 관련 개념끼리 이어지면 별자리.

- **컬러**: 딥 잉크 남색(#060a13) 베이스 + 시안(#6cd8ff)·민트(#7ef0c3) 포인트. 경고/오답은 주황·레드
- **타이포**: 헤드라인 고운바탕(세리프), UI 본문 Pretendard
- **배경**: 풀스크린 캔버스 — 반짝이는 별 150개, 성운 그라데이션, 간헐적 유성, 마우스 패럴랙스. 필름 그레인 오버레이
- **카드 시스템(공통)**: 마우스 위치를 따라 흐르는 글로우 보더 + 내부 스포트라이트. 호버한 카드만 하이라이트(3D 틸트), 나머지는 디밍
- **개념 탭**: 개념 카드 그리드 ↔ 배경 별자리 연동(호버 시 해당 별 글로우 + 관련 개념과 점선 링크). 정답률 낮은 개념은 별이 흐림. 클릭 시 스태거드 리빌 상세
- **카피 톤**: 건조하게 수치·사실만. 문학적/감성적 멘트 금지
- **목업**: `.superpowers/brainstorm/*/content/studywork-tabs-v3.html` (4개 탭 인터랙티브 데모)

## 에러 처리

- 업로드 크기 제한 (이미지 ~10MB, PDF ~20MB), 초과 시 안내
- Claude API 실패: 재시도 1회 후 사용자에게 명확한 에러 표시 (자료는 저장된 상태 유지, 추출만 재시도 가능)
- 일일 호출 상한 도달: 남은 시간 안내
- 네트워크 오류: 채팅 입력 내용 보존

## 테스트

- API 로직: Vitest (+ `@cloudflare/vitest-pool-workers`)
- 로컬 개발: `wrangler dev` (D1·R2 로컬 에뮬레이션)
- Claude 호출부는 모킹해서 프롬프트 조립·파싱 로직 검증
